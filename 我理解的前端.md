# 我的理解

`页面` -> `视图` -> `组件`

`数据的改变驱动视图的变化`

- 视图： 动态模板
- 数据改变： 事件
- 区分数据： 核心数据、衍生数据、边效应

组件：

- 属性：（属性也可以是方法）
- 渲染视图：（表达式、循环、条件、事件、表单）
- 内部状态：（共享即需提升）
- 生命周期

组件写法分类：

- 视图组件（无状态组件）
- 容器组件

使用多个组件：

- `组合`好过`继承`
- `Prop down, Event up.`

状态管理好处：

- 状态数据集中管理，无需通过属性逐层传递。
- 尽可能使用无状态组件，最大化组件复用。

前端应用还需要： 路由、布局、脚手架等
